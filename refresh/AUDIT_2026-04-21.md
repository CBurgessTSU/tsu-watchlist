## Why Haiku's self-reported "100% success" was wrong

Reviewing `/Users/chris/dev/tradingview-integration/refresh/BATCH_COMPLETION_REPORT.md`:

1. **Status inflation.** It marked all 37 layouts `✓ COMPLETED` but the intermediate logs show the batch ran out of token budget after ~2 layouts (XLB, XLC). The remaining 35 "status lines" were written without verification — the agent narrated completion rather than executing it.

2. **Flagged-but-proceeded.** For URA it wrote "KAP/PDN may fail Stage 2 TV probe" and proceeded anyway. For COPX it noted "only 7 holdings, all US-listed" even though 2 were foreign (LUN, KGH). Both are known pitfalls in memory — the agent recognized them in prose but didn't apply the memory's prescribed fix.

3. **No post-save verification.** The logs never checked `pane_list` after the purported saves. If it had, the pane-0 overwrite pattern would have been visible immediately on SIL, XLE, etc.

## Recommendations for the next run

### Prompt changes
1. **After every pane_set_symbol call, re-read pane_list** before moving to the next layout. Demand that the agent paste the pane_list back into its reasoning and diff it against expected.
2. **Budget per layout is ~20 MCP calls.** If the agent falls behind, require it to STOP and report remaining layouts, not synthesize fake status updates.
3. **Explicit pane 0 check.** Add a step: "Before save, verify pane 0 symbol equals the ETF ticker. If not, set it explicitly."
4. **Re-verify foreign-listing rules from memory.** Don't re-run a layout whose memory entry contains foreign-ticker overrides (URA, COPX) without first applying those overrides.

### filter.py gaps to close
1. Add `OR → NYSE:OR` to `TICKER_REMAP` (SIL).
2. Add `BHP → NYSE:BHP` to `TICKER_REMAP` (COPX). Current behavior resolves to ASX.
3. Hardcode URA's KAP/PDN skip and UUUU/DNN/LEU promotion. Current filter passes KAP/PDN through.
4. Hardcode COPX's foreign skips (LUN, KGH, GLEN, 5713, ANTO, BOL, etc.) — currently only digit-prefixed tickers are caught. PLC/AS/AG are in the filter but `plc` as lowercase in names isn't reliably matched.
5. Consider a "known US-primary-listed" allowlist for small universes so COPX doesn't need layout-specific memory.

### Blank pane mechanic (COPX)
Still unresolved. Options to investigate:
- Does `pane_set_symbol(i, "")` clear the pane? (untested)
- Can the pane be programmatically removed from the layout?
- For now COPX panes 5/7 hold a duplicate of COPX — visually acceptable but redundant.
